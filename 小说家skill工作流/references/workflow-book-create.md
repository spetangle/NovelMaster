# 新书创建工作流 - 含黄金三章审核

> 版本: 1.2.0 | 新增: 黄金三章专项审核
> 触发: 新建书籍时强制执行
> 用途: 在前三章完成后设置质量门槛，通过后才进入常规创作

---

## 触发条件

用户使用 `book create` 或类似指令新建书籍时自动触发。

---

## 阶段一：书籍初始化（无变化）

```
用户 → 提供创作简报（含题材、平台、目标字数）
        ↓
02_planner 读取对应题材规则
        ↓
生成 story_bible.md + book_rules.md
初始化 7 个真相文件
分配 book_id，建立目录结构
        ↓
输出创作规划书，等待用户确认
```

---

## 阶段二：前三章创作（常规流程）

```
05_writer 按序创作第1章、第2章、第3章
        ↓
06_observer 提取关键事实
        ↓
09_auditor 质量审查（常规标准：≥75分通过）
        ↓
状态更新：draft → reviewing → approved → finalized
        ↓
三章全部 finalized = true 后
        ↓
进入 阶段三：黄金三章专项审核
```

---

## 阶段三：黄金三章专项审核（新增）

### 3.1 审核触发条件

前三章全部标记为 `finalized = true` 后，自动触发专项审核。

### 3.2 审核维度与标准

| 维度 | 说明 | 通过标准 |
|------|------|----------|
| **开篇钩子** | 第1章前500字内是否有强冲突/悬念 | 必须存在 |
| **期待感建立** | 读者是否清楚主角的目标/危机/看点 | 明确 |
| **节奏密度** | 前三章平均每章有效情节事件数 | ≥3个/章 |
| **信息递进** | 每章是否有增量信息（非重复铺陈） | 递增 |
| **人设锚点** | 主角核心性格/能力是否在前三章展示 | 已建立 |
| **伏笔密度** | 埋设的伏笔是否足以支撑后续展开 | ≥2个有效伏笔 |
| **无致命缺陷** | 不存在拖沓/无聊/看不懂等问题 | 通过 |

### 3.3 审核流程

```
10_continuity_auditor 执行黄金三章审核
        ↓
生成《黄金三章质量报告》
        ↓
综合评分 = 各维度加权得分
        ↓
┌─────────────────────────────┐
│ 综合评分 ≥ 80 → 通过，进入常规创作 │
│ 综合评分 60-79 → 修订建议，可选   │
│ 综合评分 < 60 → 强制修订，三章重写 │
└─────────────────────────────┘
```

## 阶段四：决策分支

### 分支A：审核通过（≥80分）

```
输出: 「黄金三章审核通过，质量达标。」
继续创作第4章及后续章节
```

### 分支B：修订建议（60-79分）

```
输出: 「黄金三章审核发现以下问题，建议修订：」
列出问题清单
用户可选择：
  - 修订后重新审核
  - 跳过，进入常规创作（风险自负）
```

### 分支C：强制重写（<60分）

```
输出: 「黄金三章审核未通过（X分），进入强制重写流程。」
重置前三章状态为 draft
重新执行 05_writer → 06_observer → 09_auditor 流程
最多重写 3 轮
3 轮后仍不通过 → 触发人工介入
```
