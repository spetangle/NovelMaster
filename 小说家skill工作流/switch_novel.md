# 小说家 · 切换技能 (Switch Novel)

**档案编号**: SKILL-SWITCH-01  
**功能等级**: 核心系统层  
**触发条件**: 用户下达「切换当前小说」「切换小说」「当前小说改为」「激活」等指令时自动调用

---

## 1. 任务识别

识别以下关键词模式：

```
「切换当前小说至 {小说名}」
「切换小说至 {小说名}」
「当前小说改为 {小说名}」
「激活 {小说名}」
「改用 {小说名}」
「切到 {小说名}」
```

正则匹配目标：`^.*切换.*小说.*至|激活|改用|切到\s*(.+)$`

---

## 2. 数据源定位

| 文件 | 路径 | 说明 |
|-----|------|------|
| **状态文件** | `{workspace}/book_index.json` | 单一数据源，包含所有书籍索引 |
| **项目目录** | `{workspace}/books/{book_id}/` | 各书籍正文章节存放路径 |

---

## 3. 执行流程

### 步骤一：读取状态文件

读取 `book_index.json`，解析 `books` 数组中的所有书籍记录。

```json
{
  "books": [
    { "id": "chao_neng_fu_shu", "name": "超能复苏", ... },
    { "id": "tianqi_zhi_shang", "name": "天启之殇", ... }
  ]
}
```

### 步骤二：目标匹配

**完全匹配**：书籍 `name` 或 `id` 与用户输入完全一致时，直接命中。

**模糊匹配**：若完全匹配失败，检索所有书籍的 `name` 字段，取最长公共子串匹配。

**匹配失败处理**：

| 场景 | 输出 |
|-----|------|
| 目标不存在于列表中 | 「未在索引中找到《{小说名}》。当前可切换项目：{name1}、{name2}。」 |
| 多个结果命中 | 「存在多个匹配项，请确认：{选项列表}。」 |
| 无输入参数 | 「切换指令缺少目标参数。语法：切换当前小说至 {小说名}。」 |

### 步骤三：状态文件更新

锁定目标书籍后，执行以下更新：

```json
{
  "version": "1.0",
  "last_updated": "{当前时间 ISO 格式}",
  "current_novel": "{目标书籍 id}",
  "books": [ ... ]
}
```

| 字段 | 更新规则 |
|-----|---------|
| `last_updated` | 覆盖为当前时间 |
| `current_novel` | 覆盖为目标书籍 id |

### 步骤四：校验与反馈

写入完成后，读取并回显当前项目状态：

| 项目 | 值 |
|-----|---|
| 当前小说 | {name} |
| 题材 | {genre} |
| 平台 | {platform} |
| 单章字数 | {words_per_chapter} |
| 总章节 | {total_chapters} |
| 完成进度 | {completed_chapters}/{total_chapters} |

---

## 4. 输出模板

```
**切换完成。状态文件已更新。**

---

### book_index.json 更新记录

| 字段 | 变更 |
|-----|------|
| `last_updated` | {旧时间} → **{新时间}** |
| `current_novel` | ~~{旧 id}~~ → **{新 id}** |

---

### 当前项目状态锁定

| 项目 | 值 |
|-----|---|
| 当前小说 | **{name}** |
| 题材 | {genre} |
| 平台 | {platform} |
| 单章字数 | {words_per_chapter} |
| 总章节 | {total_chapters} |
| 完成进度 | {completed_chapters}/{total_chapters} |
| 项目路径 | `books/{book_id}/` |

---

变量已锁定。谋定而后动，待命。
```

---

## 5. 错误处理矩阵

| 错误类型 | 代码 | 处理方案 |
|---------|------|---------|
| 状态文件不存在 | FILE_NOT_FOUND | 创建空结构 `{"books":[]}` 并写入 |
| JSON 解析失败 | JSON_DECODE_ERROR | 备份原文件，生成新状态文件并报错 |
| 写入失败 | WRITE_ERROR | 回滚至原 `last_updated` 值并报错 |
| 目标 ID 不存在 | BOOK_NOT_FOUND | 列出所有可用书籍供用户选择 |

---

## 6. 执行约束

- 仅更新 `book_index.json`，不修改任何正文文件
- `last_updated` 使用 ISO 8601 格式：`YYYY-MM-DDTHH:mm:ss+08:00`
- 所有变更必须原子性写入，失败则回滚
- 切换完成后自动输出状态汇报，禁止静默完成
