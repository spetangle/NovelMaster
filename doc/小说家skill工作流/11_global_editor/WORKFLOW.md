# 全局内容修正工作流

> 版本: 1.0 | 适用范围: 小说家 skill 全管线
> 触发时机: 用户发起全局修正指令时执行

---

## 一、工作流概述

```
用户指令 → 解析修正类型 → 扫描目标章节 → 逐章修正 → 验证 → 更新状态
```

**修正原则**: 最小侵入——尽量不对原有叙事节奏、情感基调、伏笔结构造成影响。

---

## 二、修正类型定义

| 类型 | 触发关键词 | 修正范围 | 示例 |
|------|-----------|----------|------|
| **高频描写修正** | "反复提及"、"插入太多"、"过于频繁" | 全部章节 | 主角外貌、表情动作等 |
| **风格统一修正** | "语气不一致"、"风格不搭" | 全部章节 | 文风、句式、词汇偏好 |
| **伏笔修复修正** | "伏笔不对"、"逻辑矛盾" | 关联章节 | 前后设定不一致 |
| **角色OOC修正** | "角色崩了"、"不符合设定" | 目标角色所有出场 | 性格、语气偏差 |
| **格式规范修正** | "格式不对"、"排版问题" | 全部章节 | 标点、段落、引用格式 |

---

## 三、执行节点

### 节点0：指令解析

**执行时机**: 用户发起修正指令时

**操作内容**:
1. 识别修正类型
2. 确定修正规则（从 AGENT.md 读取或从指令中提取）
3. 确认目标范围（全部章节 / 指定章节）
4. 生成修正清单

**输出**: 修正任务单（修正类型、目标章节、修正规则）

---

### 节点1：章节扫描

**执行时机**: 修正任务单确认后

**操作内容**:
1. 按章节顺序扫描目标文件
2. 标记需要修正的位置
3. 记录修正前内容快照
4. 生成修正对照表

**修正对照表格式**:
```markdown
| 章节 | 位置 | 修正前 | 修正后 | 影响评估 |
|------|------|--------|--------|----------|
```

---

### 节点2：逐章修正

**执行时机**: 章节扫描完成后

**操作内容**:
1. 按修正规则逐章修改
2. 保持上下文连贯性
3. 避免引入新的矛盾
4. 修正后自检：读一遍，确认不破坏叙事节奏

**修正优先级**:
1. 轻微调整（词汇替换、语序调整）> 重写（段落重写）
2. 能用一句解决的，不动整个段落
3. 非核心描写优先删除，核心描写可保留

---

### 节点3：修正验证

**执行时机**: 每章修正完成后

**验证清单**:
- [ ] 修正点已落实，无遗漏
- [ ] 上下文连贯，无断层
- [ ] 字数变化在 ±5% 以内（极端情况除外）
- [ ] 无新增错别字、语病
- [ ] 伏笔结构未被破坏

---

### 节点4：状态更新

**执行时机**: 全部章节修正完成并验证通过

**操作内容**:
1. 将修正后章节同步至源文件
2. 更新 `chapter_summaries.md` 中相关章节的 `最后更新` 时间戳
3. 记录修正日志

## 四、修正规则模板

### 高频描写修正规则

```markdown
## 角色外貌描写规范

1. **首次出现原则**: 新场景/新章节首次描写即可，无需重复
2. **反应触发原则**: 仅在他人的目光、评论、或回忆触发时有选择性描写
3. **克制原则**: 避免在无关场景中插入外貌描写打断叙事节奏
4. **环境呼应**: 仅在涉及身份隐藏、伪装、或身份揭示等情节时重点描写
```

### 风格统一修正规则

```markdown
## 风格统一规范

1. **句式检查**: 避免同一句式结构连续出现超过3次
2. **词汇偏好**: 统一角色惯用词汇（如"谋定"代替"计划"、"变量"代替"变数"）
3. **语气连贯**: 确保同一角色在不同章节的对话语气一致
4. **描写密度**: 场景描写与对话比例保持均衡
```

## 五、修正效果评估

| 评估项 | 标准 | 超标处理 |
|--------|------|----------|
| 叙事节奏 | 修正后不出现明显卡顿 | 回退并重新调整 |
| 字数变化 | 单章变化不超过 ±10% | 补写或精简 |
| 伏笔完整性 | 伏笔链未断裂 | 补充伏笔说明或回退 |
| 情感基调 | 不偏离章节原本情绪 | 回退并重新调整 |
