# 伏笔管理规范参考
---

## 一、核心理念

伏笔是读者与作者的隐形契约。

埋下伏笔时，作者承诺会在未来某个时刻兑现这份期待。管理伏笔的本质，是管理这份契约的兑现节奏与质量。

**三条铁律**:
1. 每一伏笔必有其用
2. 每一伏笔必有其时
3. 每一伏笔回收必有其理

---

## 二、伏笔分类体系

### 2.1 按生命周期分类

| 类别 | 英文 | 半衰期 | 管理策略 |
|------|------|--------|----------|
| 种子伏笔 | Seed Hook | 80+ 章 | 极低频出现，逐步升温 |
| 前景伏笔 | Foreground Hook | 10-20 章 | 高频推进，保持悬念 |
| 背景伏笔 | Background Hook | 30-60 章 | 低频穿插，潜移默化 |
| 立即伏笔 | Immediate Hook | 1-2 章 | 本章埋，下章收 |

### 2.2 按重要度分类

| 等级 | 说明 | 占总伏笔比例 | 处理方式 |
|------|------|-------------|----------|
| 核心伏笔 | 支撑全书骨架 | ≤20% | 单独建档，精密管理 |
| 重要伏笔 | 影响卷内走向 | 30-40% | 常规追踪 |
| 一般伏笔 | 丰富细节层次 | 40-50% | 批量管理 |

## 三、伏笔识别训练标准

### 3.1 有效伏笔信号

以下特征出现时，应判定为伏笔：

**疑问悬念型**
- 角色对某事表示困惑或好奇
- 出现"为什么..."、"...的秘密"
- 某细节反复出现但未解释

**预言暗示型**
- 角色做出不寻常的承诺
- 出现预示性的梦境、幻象、占卜
- 某人说出"终有一天..."

**物品关联型**
- 某个物品被特别强调或赋予特殊意义
- 物品来源不明或来历蹊跷
- 物品在后续情节中有潜在用途

**身份谜题型**
- 角色身份存在模糊地带
- 某人行为与其身份不符
- 过去经历存在空白

**异常细节型**
- 某人/某事与常理相悖
- 某个细节前后矛盾
- 环境或氛围与情节不协调

### 3.2 非伏笔信号（应忽略）

以下情况不应判定为伏笔：

- 纯粹的功能性描述（"他推开门走进房间"）
- 常规互动中的礼貌用语
- 不影响情节走向的次要细节
- 已被明确解答的疑问
- 与主线完全无关的过场信息

## 四、伏笔回收质量评分标准

### 4.1 评分维度

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 逻辑自洽性 | 30% | 回收是否与伏笔暗示一致 |
| 情感冲击力 | 25% | 回收是否带来情感满足 |
| 出人意料性 | 20% | 是否超出读者预期但合理 |
| 时机恰当性 | 15% | 是否在最佳时机回收 |
| 表现手法 | 10% | 回收的叙述技巧 |

### 4.2 评分等级

| 等级 | 综合分 | 描述 |
|------|--------|------|
| 卓越 | 90-100 | 超出预期的完美回收 |
| 优良 | 80-89 | 高质量回收，超出预期 |
| 合格 | 70-79 | 符合预期的标准回收 |
| 及格 | 60-69 | 勉强达标的机械回收 |
| 不及格 | <60 | 伏笔未回收或回收失败 |

### 4.3 回收典型案例

**卓越回收示例**:
伏笔：开局描写主角手腕上的伤疤，来历不明
回收：后期揭示这是前世为拯救某人留下的印记，而这个被拯救的人正是当前反派
评价：伤疤成为连接前世与今生的情感纽带，逻辑与情感双丰收

**合格回收示例**:
伏笔：某角色多次提到"家族的使命"
回收：大结局时揭示这个使命的具体内容
评价：逻辑自洽但情感冲击力一般

**不及格回收示例**:
伏笔：开局暗示某 NPC 身份特殊
回收：后期突然揭示身份，但没有足够铺垫
评价：机械突兀，读者无法产生情感共鸣

## 五、伏笔密度管理算法

### 5.1 密度计算公式

```
章节伏笔密度 = 本章活跃伏笔数 / 本章总字数 × 10000
全书伏笔密度 = 全书活跃伏笔数 / 全书总字数 × 10000
```

**简化版（经验值）**:
```
章节伏笔密度 ≈ 本章伏笔数 × (10000 / 本章字数)
```

### 5.2 密度警戒线

| 状态 | 条件 | 处置 |
|------|------|------|
| 正常 | 1.5-4 条/章 | 无需干预 |
| 偏低 | < 1.5 条/章 | 预警：情节可能平淡 |
| 偏高 | > 4 条/章 | 预警：信息过载风险 |
| 危险 | > 6 条/章 | 强制干预：暂停新伏笔埋设 |

### 5.3 密度调节策略

**密度过低时**:
1. 在下一章增加前台伏笔埋设
2. 加速当前伏笔的推进频率
3. 引入新的短期伏笔

**密度过高时**:
1. 将部分伏笔转为"后台模式"（降低出现频率）
2. 延后非核心伏笔的埋设
3. 合并相关伏笔减少数量

## 六、伏笔链管理

### 6.1 伏笔链定义

伏笔链是由多个相互关联的伏笔组成的逻辑链条。

**结构**:
```
种子伏笔 → 推进伏笔A → 推进伏笔B → 回收伏笔
```

**示例**:
```
H001（种子）：主角身世之谜
    ↓
H010（推进）：神秘人声称认识主角的父亲
    ↓
H011（推进）：主角发现父亲留下的笔记
    ↓
H012（推进）：笔记中提到"主"的存在
    ↓
H013（回收）："主"的身份被揭示，主角身世真相大白
```

### 6.2 伏笔链管理要点

- 种子伏笔必须有至少3个推进节点
- 每个推进节点应增加新的信息量
- 推进伏笔之间的间隔不宜超过10章
- 伏笔链应有明确的最终回收节点

### 6.3 伏笔链审查清单

```
[ ] 种子伏笔是否已建立足够的推进节点
[ ] 各推进节点之间的信息增量是否充足
[ ] 推进节点间隔是否合理
[ ] 是否有节点断裂（超过10章无推进）
[ ] 最终回收节点是否已规划
```

## 七、死伏笔处理机制

### 7.1 死伏笔判定

满足以下任一条件，判定为死伏笔：

- 超过预期回收章节 3 卷仍未回收
- 与其他伏笔产生严重逻辑冲突无法调和
- 书籍完结时仍未有明确回收计划
- 伏笔核心信息已被后续情节遗忘

### 7.2 死伏笔处理方案

| 方案 | 适用情况 | 操作 |
|------|----------|------|
| 强制回收 | 书籍收尾阶段 | 在最近1-2章内强行回收 |
| 降级处理 | 伏笔重要性下降 | 降低重要度，作为细节处理 |
| 转化处理 | 伏笔可转化为日常细节 | 移除悬念，作为背景信息 |
| 合并处理 | 与另一伏笔可合并 | 合并为一个伏笔链 |
| 主动放弃 | 严重影响情节 | 从 pending_hooks.md 中移除 |

### 7.3 死伏笔处理流程

```
识别死伏笔 → 评估处理方案 → 制定处理计划 → 执行 → 验证 → 更新档案
```

## 八、伏笔管理系统集成

### 8.1 与写作管线的集成点

```
03_architect 章节规划
    ↓ 含伏笔规划
05_writer 章节创作
    ↓ 初稿完成
09_hook_manager 伏笔提取
    ↓ pending_hooks.md 更新
07_editor 评审（伏笔维度）
    ↓ 定稿
10_continuity_auditor 每5章审查
```

### 8.2 Agent 职责边界

| Agent | 职责范围 |
|-------|----------|
| 03_architect | 伏笔规划（埋设点选择） |
| 05_writer | 伏笔执行（埋设描写） |
| 09_hook_manager | 伏笔全生命周期管理 |
| 07_editor | 伏笔质量初评 |
| 10_continuity_auditor | 跨章伏笔链审查 |

### 8.3 数据流

```
pending_hooks.md ← 09_hook_manager（主维护）
                      ↑
            05_writer 提供伏笔要素
            03_architect 提供伏笔规划
            07_editor 提供评审反馈
                      ↑
            10_continuity_auditor 提供跨章审查报告
```

## 九、伏笔管理的常见错误

### 9.1 埋设类错误

| 错误类型 | 描述 | 后果 |
|----------|------|------|
| 伏而不引 | 伏笔埋下后长期不出现 | 读者遗忘，失去悬念效果 |
| 过于明显 | 伏笔伪装度不足 | 读者过早猜中，失去惊喜 |
| 过于隐蔽 | 伏笔伪装度过高 | 读者无法察觉，伏笔失效 |
| 数量失控 | 伏笔密度过高 | 信息过载，读者疲劳 |

### 9.2 回收类错误

| 错误类型 | 描述 | 后果 |
|----------|------|------|
| 有头无尾 | 伏笔埋下后不回收 | 读者产生挫败感 |
| 回收突兀 | 缺少铺垫突然揭示 | 逻辑断裂，情感断层 |
| 回收拖沓 | 回收过程过长 | 节奏拖沓，阅读疲劳 |
| 证据不足 | 伏笔要素不足以支撑回收 | 逻辑不成立 |
| 超出预期 | 回收结果与伏笔暗示矛盾 | 读者感到被骗 |

### 9.3 管理类错误

| 错误类型 | 描述 | 后果 |
|----------|------|------|
| 记录缺失 | 伏笔未登记或登记不规范 | 无法追踪 |
| 状态滞后 | 状态更新不及时 | 管理失控 |
| 密度失衡 | 全书密度波动过大 | 节奏忽快忽慢 |
| 冲突未处理 | 伏笔冲突未及时解决 | 逻辑混乱 |
