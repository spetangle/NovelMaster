# 伏笔管理标准流程

> 版本: 1.0 | 适用范围: 小说家 skill 全管线

---

## 一、伏笔管理生命周期

```
埋设规划 → 提取登记 → 状态追踪 → 回收验证 → 完结归档
```

---

## 二、管线集成节点

### 节点1：伏笔规划（03_architect 章节规划时）

**执行时机**: 03_architect 生成章节细纲时

**操作内容**:
- 在细纲中标注伏笔埋设点
- 规划本章伏笔数量（目标：2-4条）
- 标注伏笔类型与回收预期
- 检查是否与已有伏笔链冲突

**产出**: 章节细纲中含伏笔规划注释

---

### 节点2：伏笔提取（05_writer 初稿完成后）

**执行时机**: 章节初稿完成，未进入正式评审前

**操作内容**:
1. 扫描本章全文，识别伏笔要素
2. 提取伏笔类型、内容、埋设位置
3. 更新 `pending_hooks.md`
4. 计算本章伏笔密度

**伏笔识别关键词**:
- 疑问型: "为什么..."、"...的秘密"、"...的真相"
- 预言型: "将来..."、"终有一天..."、"命中注定..."
- 物品型: "这枚..."、"这把..."、"这块..."
- 异常型: "不应该..."、"反常..."、"矛盾..."

**判定标准**:
| 识别特征 | 判定 |
|----------|------|
| 明确指向后续情节 | 有效伏笔 |
| 制造悬念或疑问 | 有效伏笔 |
| 仅功能性描述无悬念 | 非伏笔 |
| 与已有伏笔完全重复 | 合并处理 |

---

### 节点3：伏笔状态更新（章节定稿后）

**执行时机**: 章节通过质量评审后

**操作内容**:
1. 将本章新增伏笔标记为"埋设中"
2. 将推进中的伏笔标记为"推进中"
3. 将已回收伏笔标记为"已回收"
4. 同步更新更新日志

---

### 节点4：伏笔回收验证（伏笔预期回收章完成时）

**执行时机**: 伏笔预期回收章节定稿后

**验证清单**:
- [ ] 伏笔核心问题是否被回答
- [ ] 伏笔回收是否有足够铺垫（至少2章前开始回收）
- [ ] 回收方式是否合理（非机械回应）
- [ ] 伏笔回收后是否留下余韵（可转化为后续伏笔）

**回收评分标准**:
| 评分 | 描述 |
|------|------|
| 优良 | 回收出人意料又情理之中，有情感冲击力 |
| 合格 | 伏笔问题被完整回答，逻辑自洽 |
| 不足 | 回收过于机械，缺少情感铺垫 |
| 失败 | 伏笔未回收或回收时证据不足 |

---

### 节点5：伏笔链联合审查（每5章 + 10_continuity_auditor）

**执行时机**: 每5章完成时

**联合审查清单**:
1. 伏笔链是否完整闭合
2. 伏笔密度分布是否合理
3. 长线伏笔是否有推进信号
4. 伏笔之间是否有逻辑矛盾
5. 是否有过期未推进的"死伏笔"

**死伏笔处理**:
- 超过预期回收章节3卷未回收 → 标记为"待处理"
- 评估是否需要强制回收或降级处理
- 更新伏笔档案，注明处理方案

## 三、伏笔密度控制

### 密度标准

| 书籍类型 | 推荐密度 | 峰值密度（高潮卷） | 低谷密度（过渡卷） |
|----------|----------|-------------------|-------------------|
| 长篇（300+章） | 2-3条/章 | 4-5条/章 | 1-2条/章 |
| 中篇（50-300章） | 3-4条/章 | 5-6条/章 | 2条/章 |
| 短篇（<50章） | 4-5条/章 | 6-8条/章 | 3条/章 |

### 密度异常处理

```
密度 < 1条/章 → 预警：情节平淡，增加伏笔或加速主线
密度 > 6条/章 → 预警：信息过载，延迟部分伏笔埋设
密度突增 > 100% → 预警：检查是否与前章伏笔重叠
```

## 四、伏笔分类管理规范

### 4.1 种子伏笔（开局型）

- **特征**: 开局即埋，贯穿全书，回收周期极长
- **管理要求**: 
  - 初期展示频率低（5-10章出现1次）
  - 中期逐步增加出现频率
  - 后期成为核心悬念
- **半衰期**: 80+章
- **示例**: 主角身世之谜、远古预言

### 4.2 前台伏笔（情节推进型）

- **特征**: 与当前情节线强关联，推动读者追更
- **管理要求**:
  - 每章至少1条在推进
  - 连续2章无推进则预警
- **半衰期**: 10-20章
- **示例**: 反派阴谋线索、角色关系变化

### 4.3 后台伏笔（暗线型）

- **特征**: 不抢主线视线，潜移默化
- **管理要求**:
  - 出现时以日常化描写掩护
  - 推进频率：每3-5章1次
- **半衰期**: 30-60章
- **示例**: 幕后势力布局、自然灾害前兆

### 4.4 立即伏笔（悬念型）

- **特征**: 本章埋，下章收，制造强悬念
- **管理要求**:
  - 密度不超过总伏笔的20%
  - 回收必须在本章有明确回应
- **半衰期**: 1-2章
- **示例**: 角色死亡假象、宝物突然出现

## 五、伏笔冲突检测规则

### 5.1 冲突类型

| 冲突类型 | 描述 | 严重度 |
|----------|------|--------|
| 逻辑冲突 | 两个伏笔暗示相互矛盾的结论 | 高 |
| 时间冲突 | 同一伏笔在同一时间有两种状态 | 高 |
| 因果冲突 | 伏笔A的实现会导致伏笔B无法成立 | 中 |
| 密度冲突 | 密集伏笔群集中在同一章节 | 低 |

### 5.2 冲突处理流程

```
检测到冲突 → 评估严重度 → 制定处理方案 → 执行 → 验证
```

**处理方案选项**:
1. 合并：将两个相关伏笔合并为1个
2. 分解：将1个复杂伏笔分解为多个独立伏笔
3. 延后：调整其中一个伏笔的埋设时间
4. 放弃：删除影响较小的伏笔

## 七、伏笔管理系统检查清单

### 写作前
- [ ] 章节细纲含伏笔规划
- [ ] 伏笔数量在合理区间
- [ ] 无与已有伏笔链冲突

### 写作中
- [ ] 伏笔埋设点已按计划执行
- [ ] 伏笔密度无异常波动

### 初稿后（09_hook_manager 介入）
- [ ] 伏笔已正确提取
- [ ] pending_hooks.md 已更新
- [ ] 密度报告已生成

### 定稿前
- [ ] 预期回收伏笔已正确回收
- [ ] 回收质量评分≥合格

### 定稿后
- [ ] 状态变更已同步
- [ ] 更新日志已记录

### 每5章（联合审查）
- [ ] 伏笔链完整性检查
- [ ] 死伏笔处理
- [ ] 密度分布分析
